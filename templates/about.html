FHIR Demo

FHIR are the newest records used for transferring medical records. FHIR = Fast Healthcare Interoperability Resources. They were created by HL7 and are made for RESTful API calls.

For all the good that FHIR brings, it also brings up several problems. Some of these are that all key fields are NOT required to be present, it is extensible which allows vendors to add fields, and lastly it must be canonical.

FHIR.org has mounted a server that is available on-line and it purposefully attempts to break your record handling. This demo calls that server and demonstrates that with proper error handling, you can avoid crashing.

Technologies used:
- Python language
- Flash web server
- Simple HTML pages
- API calls
- FHIR server calls
- FHIR record handling
- FHIR error handling
